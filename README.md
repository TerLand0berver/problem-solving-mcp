# 问题解决MCP服务器

一个基于Model Context Protocol (MCP)的智能问题解决系统，能够根据问题复杂度自动创建3-12个不同职能的角色团队，采用重要紧急4分法进行任务管理，实现多角色协同和并行处理的高效解决方案。

## 🌟 核心特性

### 🎭 智能角色创建
- **角色创建者**：根据问题复杂度（1-10分）自动创建3-12人专业团队
- **12种专业角色**：系统分析师、研究员、设计师、开发工程师、测试工程师、项目经理、领域专家、战略规划师、沟通协调员、质量保证专员、风险管理专员、创新专员
- **智能匹配**：根据问题领域和约束条件自动选择最适合的角色组合

### 🔍 结果检查者
- **多维度评估**：完整性、可行性、质量、风险、时间线五个维度
- **问题识别**：自动识别关键、高、中、低四个级别的问题
- **改进建议**：提供具体的优化建议和修复方案
- **质量把关**：确保解决方案达到批准标准

### 🎯 协调者
- **全流程管理**：从问题分析到方案执行的完整协调
- **自动改进**：方案不合格时自动进行优化迭代
- **执行监控**：模拟和跟踪解决方案执行过程
- **反思总结**：每个阶段进行经验总结和教训提取

### ⚡ 重要紧急4分法
- **第一象限**：紧急且重要 - 立即处理
- **第二象限**：重要但不紧急 - 计划处理  
- **第三象限**：紧急但不重要 - 委派处理
- **第四象限**：既不紧急也不重要 - 消除或推迟

### 🔄 并行处理优化
- **任务依赖分析**：自动识别任务间的依赖关系
- **并行度优化**：最大化可并行执行的任务数量
- **资源平衡**：智能分配角色工作负载
- **效率提升**：通过并行处理显著缩短项目周期

## 🛠️ 可用工具

### 核心功能
- `create_problem` - 创建待解决问题
- `solve_problem` - 完整解决问题流程
- `get_role_recommendations` - 获取角色配置建议
- `check_solution` - 检查解决方案质量

### 团队管理
- `get_team_status` - 查看团队状态和进度
- `update_task_status` - 更新任务状态
- `add_reflection` - 添加团队反思记录

### 分析工具
- `eisenhower_matrix_analysis` - 重要紧急4分法分析
- `analyze_parallelism` - 任务并行性分析
- `optimize_task_allocation` - 优化任务分配
- `assess_risks` - 风险评估

### 报告功能
- `generate_progress_report` - 生成进度报告
- `get_problem_history` - 查看历史记录
- `simulate_execution` - 模拟执行过程

## 📝 使用示例

### 1. 创建问题
```json
{
  "tool": "create_problem",
  "arguments": {
    "title": "开发AI客服系统",
    "description": "为公司开发一个智能客服系统，能够处理常见客户咨询，提供24/7服务",
    "domain": "人工智能技术开发",
    "complexity_score": 8,
    "constraints": ["预算限制", "3个月交付", "需要支持多语言"],
    "success_criteria": ["客户满意度85%以上", "响应时间<2秒", "准确率90%以上"],
    "stakeholders": ["产品经理", "技术总监", "客服部门", "用户"]
  }
}
```

### 2. 获取角色建议
```json
{
  "tool": "get_role_recommendations", 
  "arguments": {
    "problem_id": "问题ID"
  }
}
```

### 3. 解决问题
```json
{
  "tool": "solve_problem",
  "arguments": {
    "problem_id": "问题ID"
  }
}
```

### 4. 分析任务优先级
```json
{
  "tool": "eisenhower_matrix_analysis",
  "arguments": {
    "team_id": "团队ID"
  }
}
```

## 🔧 配置说明

### 角色模板
系统内置12种专业角色模板，每个角色包含：
- 名称和类型
- 详细描述
- 核心技能列表
- 主要职责
- 专业水平评级

### 优先级策略
采用重要紧急4分法：
- **紧急且重要**：立即处理，优先级最高
- **重要但不紧急**：计划安排，重点关注
- **紧急但不重要**：委派处理，避免干扰
- **既不紧急也不重要**：消除或推迟

### 质量标准
- **完整性**：方案要素齐全，覆盖全面
- **可行性**：资源配置合理，技能匹配
- **质量**：文档规范，描述清晰
- **风险**：识别充分，缓解措施完善
- **时间线**：估算合理，并行度高
